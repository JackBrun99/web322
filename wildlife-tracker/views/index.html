Project Description

The Wildlife Sighting Tracker is a web application designed to help nature enthusiasts, researchers, and conservationists record and manage wildlife observation data. This application serves as a digital logbook for tracking various species sightings across different habitats.

Key Features:

  • Data Management: Store and organize wildlife sighting records with species information, locations, dates, and verification status
  • Advanced Filtering: Filter sightings by verification status, habitat type, or specific species
  • Search Capabilities: Quickly find specific sightings using intelligent search functions
  • API Access: Complete RESTful API for programmatic access to the wildlife data
  • Responsive Design: Accessible on all devices from desktop to mobile

Technical Implementation:

Built with Express.js, this application demonstrates modern web development practices including asynchronous operations, RESTful API design, data filtering and manipulation, and responsive front-end development. The backend efficiently handles data operations while providing a clean, user-friendly interface.

API Endpoints

Click on any endpoint below to test it in your browser!

  • GET /api/sightings

    Returns all wildlife sighting records from the database.

  • GET /api/sightings/verified

    Filters and returns only the verified sightings (where verified status is true).

  • GET /api/sightings/species-list

    Extracts and returns an array of unique species names from all sightings.

  • GET /api/sightings/habitat/forest

    Returns all sightings from forest habitats with count statistics.

  • GET /api/sightings/search/eagle

    Searches for sightings containing "eagle" in species name (case-insensitive).

  • GET /api/sightings/find-index/moose

    Finds the index position of moose sightings in the database.

  • GET /api/sightings/recent

    Returns the three most recent sightings based on date.
